Comparación de código — almacenamiento de contraseñas

❌ Vulnerable — MD5 sin sal

// MD5: rápido, sin sal, deprecated const md5 = s => crypto .createHash('md5') .update(s) .digest('hex'); // Almacena: 482c811da5d5b4bc6d497ffa98491e38 // Crackeable en <1ms con rainbow table // hashcat puede hacer 10B+ hash/seg con GPU user.hash = md5(password); // Verificación: md5(input) === user.hash // reversible

✅ Segura — bcrypt con sal (cost=12)

// bcrypt: adaptativo, salted, lento por diseño const bcrypt = require('bcryptjs'); // Almacena: $2b$12$N9qo8uLOi... // Incluye sal única + cost factor // ~250ms por hash (ajustable) user.hash = await bcrypt.hash(password, 12); // Verificación: await bcrypt.compare(input, user.hash) // sal y cost extraídos del hash automáticamente // no reversible con rainbow tables
Por qué bcrypt: el cost factor (12) hace que cada verificación tome ~250ms. Con 10 mil millones de hashes/seg (GPU), crackear una contraseña bcrypt de 8 caracteres tomaría miles de años. MD5 se crackea en segundos.
