Comparación de código — merge vulnerable vs. seguro

❌ custom-merge v1.0 (vulnerable — CVE-2019-XXXX)

// Simula lodash < 4.17.12 function vulnerableMerge(target, source) { for (const key in source) { // ← itera __proto__ if (typeof source[key] === 'object') { if (!target[key]) target[key] = {}; // target['__proto__'] = Object.prototype // ← recursivo sobre Object.prototype! vulnerableMerge(target[key], source[key]); } else { target[key] = source[key]; } } }

✅ custom-merge v2.1 (parcheado)

// Bloquea claves peligrosas function safeMerge(target, source) { const BLOCKED = new Set([ '__proto__', 'constructor', 'prototype' ]); for (const key of Object.keys(source)) { if (BLOCKED.has(key)) continue; // ← bloqueado if (typeof source[key] === 'object') { target[key] = Object.create(null); safeMerge(target[key], source[key]); } else { target[key] = source[key]; } } }
Supply Chain Risk: El mismo patrón afectó a lodash (CVE-2019-10744), minimist (CVE-2020-7598), jQuery (CVE-2019-11358), Shai-Hulud npm worm (2025). Usar paquetes desactualizados introduce vulnerabilidades de terceros en tu código.
